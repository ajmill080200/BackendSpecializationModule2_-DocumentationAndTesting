Backend Specialization Module 2 - Documentation and Testing

Backend Specialization Module 2 Documentation and Testing - API with Application Factory Pattern is building out a small mechanic shop application using MySQL local database connections, CRUD API end points, and organizing the application using the Application Factory Pattern structure.  The API is also using Flask-Limiter, Flask-Caching, token authentication, junction table, and pagination.  Documentation and unit testing added in to help developers understand all end points available.

The small mechanic shop application allows the user to add/edit/remove/view customers, add/edit/remove/view mechanics, add/edit/remove/view service tickets, assign/remove service tickets to/from mechanics, rank mechanics by ticket count, add/edit/remove/view inventory and assign inventory to service tickets.

Enjoy the Mechanic Shop App!
